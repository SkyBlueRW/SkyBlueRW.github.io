## What this is

A personal Jekyll blog ("Ray's Hub") on quantitative finance — portfolio construction, asset pricing, factor models, statistical methods. Published at https://skybluerw.github.io using the **Chirpy** theme (`jekyll-theme-chirpy`, a Ruby gem). The tone is casual/personal, not an academic journal.

## Build & deploy

- **Theme is a gem** (see `Gemfile`), so you need Ruby **3.x** (Chirpy 7 does not support the system Ruby 2.6). Install via Homebrew/rbenv, then:
  ```bash
  bundle install
  bundle exec jekyll s        # local preview at http://localhost:4000
  bundle exec jekyll b        # build into _site/
  bundle exec htmlproofer _site --disable-external   # link check (same as CI)
  ```
- **Deployment is via GitHub Actions**, not the legacy branch build. `.github/workflows/pages-deploy.yml` builds with Ruby 3.4 + htmlproofer and publishes through the Pages "GitHub Actions" source. Pushing to `main` deploys. The repo's **Settings → Pages → Source must be set to "GitHub Actions"** (one-time manual step).

## Authoring posts (key conventions)

- Posts live in `_posts/` named `YYYY-MM-DD-slug.md`; **post URLs are `/posts/:title/`** (set by `permalink` in `_config.yml` defaults — note this differs from the old `/YYYY/MM/DD/slug.html` scheme).
- **Every post needs YAML front matter** (Chirpy requires it):
  ```yaml
  ---
  title: 'Post Title'
  date: 2024-06-26 12:00:00 -0500
  categories: [Asset Pricing]        # used by the Categories tab
  tags: [factor-zoo, bayesian]       # lowercase; used by the Tags tab
  math: true                         # ONLY on posts with LaTeX — loads MathJax
  ---
  ```
- **Categories in use** (keep consistent so the Categories page stays clean): `Portfolio Construction`, `Asset Pricing`, `Factor Models`, `Dev`.
- **Math:** set `math: true` in front matter (Chirpy loads MathJax per-post on demand); then write LaTeX directly. Do not add MathJax script tags.
- **TOC** is auto-generated by Chirpy from the post's `##`/`###` headings (`toc: true` is the global default) — do not hand-write tables of contents. Images are referenced via absolute `raw.githubusercontent.com/.../_posts/asset/...` URLs.
- A post dated in the future stays hidden until its date unless `future: true` is set in `_config.yml`.

## Site structure

- `_config.yml` — Chirpy config: site identity, `avatar`, `social.links` (author/copyright), timezone, permalink, `jekyll-archives`. Do not touch the "not recommended to modify" block (kramdown/rouge, collections, sass) without reason.
- `_data/contact.yml` — the contact icons at the bottom of the sidebar (github, email, linkedin, rss).
- `_tabs/` — left-sidebar nav entries; front-matter-only stubs whose `layout`/`icon`/`order` drive the menu. `about.md` is the only one with real content; `categories.md`/`tags.md`/`archives.md` use Chirpy's built-in layouts. **Categories/tags pages are generated automatically** — there are no hand-curated nav pages anymore.
- `_plugins/posts-lastmod-hook.rb` — sets each post's last-modified date from git history.
- `assets/img/avatar.svg` — placeholder sidebar avatar (initials). Replace with a real photo and update `avatar:` in `_config.yml`.
- Layouts/includes/CSS/JS all come from the Chirpy gem (not vendored here), so there are no local `_layouts/` or `_includes/` to edit. To override theme files, copy the specific file from the gem into a same-named local path.
